# Codex Prompts

## One-Sentence Summary

Implementation prompts for turning theory into working software.

## Why It Matters

Good prompts prevent vague finance output and force useful structure.

## Core Idea

```txt
Explain [theory] with assumptions, formula, simple example, limitation, and a software project.
```

```txt
Build a beginner-readable Python tool for [theory]. Use sample data, no paid APIs, and run a syntax check.
```

```txt
Compare [theory A] vs [theory B]. State what each explains, where each fails, and how an agent should use them.
```

## Key Terms

- Prompt
- Assumption
- Formula
- Evidence
- Verification

## Framework

Define inputs, outputs, constraints, sample data, and the smallest useful check.

## Simple Example

Instead of asking if a stock is cheap, ask which model says it is cheap and what could stop correction.

## Real-World Use

Use with Codex, research agents, and market-analysis workflows.

## Limitations

- Prompts do not replace source verification.
- Agents can sound certain without data.

- Prompt library UI
